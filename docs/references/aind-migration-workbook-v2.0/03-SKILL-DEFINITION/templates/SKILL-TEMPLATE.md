# {Skill Name}

> **CRITICAL - 파일 위치**: 이 파일은 반드시 `.claude/skills/{name}/SKILL.md` 경로에 저장해야 합니다.
> - 디렉토리명 = `name` 필드 (정확히 일치 필수!)
> - **권장 명명 규칙**: `s{stage}-{phase}-{workflow}-{action}`
> - 예: `name: s1-01-discovery-feature-inventory` → `.claude/skills/s1-01-discovery-feature-inventory/SKILL.md`
> - **중첩 구조 금지**: `.claude/skills/discovery/feature-inventory/` (❌)

> **CRITICAL**: Frontmatter(위 `---` 블록)는 반드시 파일 맨 처음에 raw YAML로 시작해야 합니다.
> 코드 블록(` ```yaml `)으로 감싸면 Claude Code가 인식하지 못합니다.

> **Frontmatter 규칙** (Claude Code 표준)
> - `name`: 문자, 숫자, 하이픈만 사용 (특수문자 금지, **디렉토리명과 일치 필수**)
> - `description`: "Use when..."으로 시작, 트리거 조건만 기술, 워크플로우 요약 금지
> - 최대 1024자
>
> **나쁜 예시:** `description: Use when executing plans - dispatches subagent per task with code review`
> **좋은 예시:** `description: Use when executing implementation plans with independent tasks in the current session`

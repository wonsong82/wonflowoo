## 2. 인프라 준비

### 2.1 교육장 환경

| 항목 | 요구사항 | 체크 |
|------|---------|------|
| 좌석 | 수강생 인원 + 여유 20% | [ ] |
| 전원 콘센트 | 좌석당 1개 이상 | [ ] |
| 네트워크 | 유선/WiFi, 수강생당 최소 10Mbps | [ ] |
| 프로젝터/대형 모니터 | 강사용 1대 + 발표용 1대 | [ ] |
| 화이트보드 | 최소 1개 (워크샵용) | [ ] |

### 2.2 실습 서버 환경

```yaml
server_requirements:
  # Option A: 클라우드 기반
  cloud:
    provider: "AWS/GCP/Azure"
    instance_type: "개발용 인스턴스"
    count: "수강생 수 또는 공유"

  # Option B: 로컬 환경
  local:
    os: "macOS / Linux / Windows 10+"
    memory: ">= 16GB RAM (권장)"
    storage: ">= 50GB 여유 공간"

  software:
    required:
      - "Git >= 2.30"
      - "Node.js >= 18 (Claude Code용)"
      - "Python >= 3.11 (Orchestrator용)"
      - "Java 11+ (레거시가 Java인 경우)"
    optional:
      - "Docker (환경 격리용)"
      - "VS Code + 확장"
```

### 2.3 네트워크 요구사항

```yaml
network_requirements:
  external_access:
    - "api.anthropic.com (Claude API)"
    - "github.com (코드베이스)"
    - "npm registry (패키지)"

  bandwidth:
    minimum: "수강생당 10Mbps"
    recommended: "수강생당 50Mbps"

  firewall:
    - "HTTPS (443) 아웃바운드 허용"
    - "SSH (22) 필요시 허용"
```

### 3.2 실습 레거시 코드베이스

```yaml
codebase_selection:
  option_1_internal:
    description: "회사 내부 실제 레거시 시스템"
    pros:
      - "가장 실전적"
      - "실무 적용 즉시 가능"
    cons:
      - "보안/접근 권한 이슈"
      - "복잡도가 너무 높을 수 있음"
    preparation:
      - "접근 권한 신청"
      - "민감 정보 마스킹"
      - "적절한 범위 선정"

  option_2_sanitized:
    description: "실제 레거시의 익명화된 버전"
    pros:
      - "실전적이면서 보안 이슈 없음"
    cons:
      - "준비 작업 필요"
    preparation:
      - "민감 정보 제거"
      - "도메인명/변수명 익명화"
      - "테스트 데이터 생성"

  option_3_opensource:
    description: "오픈소스 레거시 프로젝트"
    examples:
      - "Spring PetClinic (소규모)"
      - "Broadleaf Commerce (중규모)"
      - "OpenMRS (대규모)"
    pros:
      - "접근 용이"
      - "보안 이슈 없음"
    cons:
      - "회사 실무와 차이"
```

## 6. 수강생 관리

### 6.1 수강생 선발 기준

```yaml
selection_criteria:
  required:
    - "개발 경력 3년 이상 (중급 이상)"
    - "Git 기본 사용 가능"
    - "CLI 환경 익숙"

  preferred:
    - "Java/Spring 경험"
    - "레거시 마이그레이션 관련 경험"
    - "AI 도구 사용 경험"

  group_size:
    minimum: 8
    maximum: 20
    optimal: 12-16
```

### 6.2 사전 안내 (D-7)

수강생에게 발송할 사전 안내 내용:

```markdown
# AIND Migration 교육 사전 안내

## 교육 일정
- 일시: YYYY-MM-DD ~ YYYY-MM-DD (10일간)
- 장소: [교육장 주소]
- 시간: 09:00-18:00

## 준비물
- [ ] 노트북 (개발 가능한 환경)
- [ ] 충전기
- [ ] 필기구

## 사전 설치 (선택)
아래 소프트웨어를 미리 설치하시면 Day 1 실습이 원활합니다:
- Git: https://git-scm.com/
- Node.js 18+: https://nodejs.org/
- VS Code: https://code.visualstudio.com/

## 사전 학습 (선택)
- Claude Code 소개: [링크]
- 워크북 Overview 읽기: [링크]

## 문의
- 담당자: [이름] ([이메일])
```

### 6.3 사전 설문

교육 전 수강생 수준 파악을 위한 설문:

```yaml
pre_survey:
  background:
    - "현재 역할/직무는?"
    - "개발 경력은?"
    - "주 사용 언어/프레임워크는?"

  experience:
    - "레거시 마이그레이션 경험이 있습니까?"
    - "AI 코딩 도구(Copilot, Claude 등) 사용 경험은?"
    - "현재 맡고 있는 레거시 시스템이 있습니까?"

  expectation:
    - "이 교육에서 가장 배우고 싶은 것은?"
    - "교육 후 바로 적용할 프로젝트가 있습니까?"
```

## 8. 비상 대응 계획

### 8.1 기술적 이슈

| 상황 | 대응 방안 |
|------|----------|
| 네트워크 장애 | 모바일 핫스팟 준비, 오프라인 자료 활용 |
| Claude API 장애 | 사전 녹화된 데모 영상 활용, 이론 시간 조정 |
| API 한도 초과 | 백업 API 키 준비, 공유 계정 활용 |
| 실습 서버 장애 | 로컬 환경 대안 안내, 페어 실습 |

### 8.2 운영 이슈

| 상황 | 대응 방안 |
|------|----------|
| 강사 불참 | 백업 강사 지정, 일정 조정 |
| 수강생 중도 이탈 | 따라잡기 자료 제공, 개별 보충 |
| 진도 지연 | 선택 주제 조정, 자습 자료 제공 |
| 수준 차이 심함 | 팀 재편성, 보조 강사 집중 지원 |

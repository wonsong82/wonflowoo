## 개요

Claude Skills는 **반복되는 작업 흐름을 한 번 정의해 지속적으로 재사용**하도록 설계된 워크플로 지식 패키지.

- 앤트로픽이 직접 작성한 33p PDF 가이드
- 스킬 설계 → 구조화 → 테스트 → 배포 전 과정
- 상위 2~3개 워크플로가 정리되어 있다면 **15~30분 내 첫 스킬 구축 가능**

## Chapter 1: 기본 개념 (Fundamentals)

### Skill이란?

Claude에게 **특정 작업이나 워크플로를 수행하는 방법을 한 번 가르쳐 반복 사용**할 수 있게 하는 구조.

**효과적인 상황:**
- 스펙 기반 프론트엔드 디자인 생성
- 일관된 방식의 리서치 수행
- 팀 스타일 가이드를 따르는 문서 작성
- 다단계 프로세스 자동 실행

### Skill 기본 구성

```
skill-name/
├── SKILL.md (필수)
│   ├── YAML frontmatter (name, description)
│   └── Markdown 지침
└── 선택 구성
    ├── scripts/     - 실행 코드 (Python/Bash)
    ├── references/  - 참조 문서
    └── assets/      - 템플릿, 리소스
```

### 핵심 설계 원칙

#### 1. Progressive Disclosure (3단계 정보 로딩)

| 단계 | 내용 | 로딩 시점 |
|------|------|----------|
| 1단계 | YAML frontmatter | 항상 (시스템 프롬프트) |
| 2단계 | SKILL.md 본문 | 스킬이 관련 있다고 판단 시 |
| 3단계 | references/scripts/assets | 필요 시에만 |

#### 2. Composability
- 여러 Skills 동시 로드 가능
- 단독 실행 가정 X, 충돌 방지 설계

#### 3. Portability
- Claude.ai, Claude Code, API 환경 동일 동작
- 플랫폼별 수정 없이 재사용

### MCP와 Skills의 관계

| 요소 | 역할 | 비유 |
|------|------|------|
| MCP | 도구와 데이터 접근 제공 | 주방, 재료, 조리 도구 |
| Skills | 도구 사용 방법 정의 | 레시피 |

> MCP가 "무엇을 할 수 있는지"를 제공한다면, Skills는 "어떻게 해야 하는지"를 고정

## Chapter 2: 계획과 설계 (Planning and Design)

### 출발점: 사용 사례 정의

스킬 작성 전 **2~3개의 구체적인 사용 사례** 정의 필수.

**좋은 사용 사례 구성 요소:**
1. 사용자가 달성하고자 하는 목표
2. 사용자가 말할 법한 트리거 문장
3. 내부적으로 수행해야 할 단계적 작업
4. 사용되는 도구
5. 최종 결과 상태

### Skill 사용 사례 3가지 유형

| Category | 설명 | 특징 |
|----------|------|------|
| **1. 문서/자산 생성** | 일관된 품질의 산출물 | 스타일 가이드 내장, 템플릿 활용, Claude 기본 기능만으로 완결 |
| **2. 워크플로 자동화** | 반복 프로세스 | 단계별 검증, 중간 리뷰 루프, 과정의 안정성 중시 |
| **3. MCP 강화** | MCP 도구를 워크플로로 전환 | 여러 MCP 호출 조합, 오류 처리 내장 |

### YAML frontmatter 설계

```yaml
---
name: skill-name
description: "스킬 설명 (1024자 이내)"
---
```

**description 필수 포함 내용:**
- 스킬이 무엇을 하는지
- 언제 사용하는지
- 사용자가 말할 법한 구체적 표현

## Chapter 3: 테스트와 반복 개선

### 테스트 수준 선택

| 수준 | 환경 | 적합한 경우 |
|------|------|------------|
| 수동 테스트 | Claude.ai | 초기 설계 검증, 빠른 수정 |
| 스크립트 테스트 | Claude Code | 회귀 테스트, 내부 팀용 |
| API 테스트 | Skills API | 대규모 배포, 엔터프라이즈 |

### 핵심 테스트 영역

1. **트리거 테스트**: 적절한 상황에서만 자동 로드되는지
2. **기능 테스트**: 의도한 결과를 정확히 생성하는지
3. **성능 비교 테스트**: 스킬 사용 전후 개선 효과

### 피드백 신호와 해결책

| 신호 | 원인 | 해결책 |
|------|------|--------|
| 트리거 부족 | description이 추상적 | 구체적 표현/용어 추가 |
| 트리거 과다 | 범위가 너무 넓음 | 범위 축소, 부정 트리거 추가 |
| 실행 문제 | 지침 불명확 | 지침 명확화, 오류 처리 강화 |

## Chapter 4: 배포와 공유

### 배포 방식 (2026년 1월 기준)

**개인 사용자:**
1. Skill 폴더 다운로드
2. Claude.ai > Settings > Capabilities > Skills 업로드
3. 또는 Claude Code의 skills 디렉터리에 배치

**조직 단위:**
- 관리자가 워크스페이스 전체에 배포
- 중앙 집중식 관리 + 자동 업데이트

### API를 통한 Skills 활용

- `/v1/skills` 엔드포인트로 스킬 관리
- Messages API에서 `container.skills` 파라미터로 지정
- Claude Agent SDK와 연동 가능

## Chapter 5: 패턴과 문제 해결

### 설계 패턴 5가지

| 패턴 | 설명 | 적합한 경우 |
|------|------|------------|
| **순차 워크플로** | 정해진 순서로 단계 수행 | 온보딩, 계정 생성 |
| **다중 MCP 협업** | 여러 서비스 연속 사용 | 디자인→저장→태스크→알림 |
| **반복 개선** | 초안→검증→수정 루프 | 보고서, 문서 품질 개선 |
| **상황 인식 도구 선택** | 조건에 따라 최적 도구 선택 | 스토리지, 문서 관리 |
| **도메인 특화 지능** | 전문 지식과 규칙 내재화 | 금융, 컴플라이언스, 보안 |

### 문제 해결 체크리스트

| 문제 | 원인 | 해결 |
|------|------|------|
| 업로드 실패 | SKILL.md 파일명, YAML 형식 오류 | 파일명 확인, YAML 검증 |
| 트리거 안 됨 | description 추상적 | 사용자 표현 반영 |
| MCP 호출 실패 | 연결/인증 문제 | MCP 단독 호출로 분리 테스트 |
| 지침 무시 | 지침 장황, 핵심 묻힘 | 중요 조건 상단 배치, 스크립트화 |
| 성능 저하 | SKILL.md 과대 | references로 분리, 스킬 수 축소 |

## 핵심 요약

> "Skills는 프롬프트가 아니라 **재사용 가능한 작업 지식**"

1. **문제 중심 설계**: 도구가 아닌 결과에 초점
2. **Progressive Disclosure**: 필요한 정보만 로드
3. **반복 개선**: 한 번에 완성이 아닌 지속적 다듬기
4. **MCP + Skills = 완전한 자동화**: 도구 접근 + 사용법

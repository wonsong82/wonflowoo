## 핵심 요약

StrongDM AI팀이 제안하는 **Software Factory** 개념: 코드를 사람이 보지 않고도 고품질 소프트웨어를 만드는 비대화형 개발 방식. 명세와 시나리오 기반으로 에이전트가 코드를 작성하고, 테스트 하네스로 검증하며, 인간 검토 없이 자동 수렴한다. 엔지니어당 하루 $1,000+ 토큰 비용이 필요.

---

## Software Factory 핵심 원칙

### 손 떼기 (Hands Off)
- **코드는 사람이 직접 작성하면 안 됨**
- 인간의 코드 작성 및 리뷰 금지
- 모든 개발 과정을 에이전트가 수행

### 비대화형 개발
- 명세(specs)와 시나리오(scenarios)가 에이전트를 구동
- 코드 작성 → 검증 → 수렴까지 자동화
- Claude 3.5 (2024년 10월 개정판)부터 "정확도 복리 축적" 가능해짐

---

## 테스트의 진화: 시나리오와 만족도

### 기존 테스트의 한계
- 에이전트가 `return true`로 테스트만 통과시키는 지름길 선택
- 좁게 작성된 테스트는 실제 소프트웨어로 일반화되지 않음
- 코드에 맞춰 테스트가 게으르게 재작성될 수 있음

### 새로운 개념

| 개념 | 설명 |
|------|------|
| **시나리오** | 엔드투엔드 사용자 스토리, 코드베이스 외부에 저장 (홀드아웃 세트처럼) |
| **만족도** | 불리언 대신 확률적 평가 - 사용자를 만족시킬 가능성의 비율 |

## Digital Twin Universe (DTU)

서드파티 서비스의 **행동 클론**을 만들어 대규모 검증 수행:

- **복제 대상**: Okta, Jira, Slack, Google Docs/Drive/Sheets
- **장점**:
  - 프로덕션 한계를 초과하는 볼륨/속도로 검증
  - 위험한 실패 모드 테스트 가능
  - API 비용 없이 시간당 수천 개 시나리오 실행
  - 결정적이고 재생 가능한 테스트 조건

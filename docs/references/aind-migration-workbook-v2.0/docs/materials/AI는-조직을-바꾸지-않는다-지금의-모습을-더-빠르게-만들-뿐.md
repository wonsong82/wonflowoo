# AI는 조직을 바꾸지 않는다, 지금의 모습을 더 빠르게 만들 뿐

> **출처**: [AI Sparkup](https://aisparkup.com/posts/9659)

> **참고 보고서**: [The GenAI Divide – State of AI in Business 2025](https://mlq.ai/media/quarterly_decks/v0.1_State_of_AI_in_Business_2025_Report.pdf) — MIT Project NANDA

## 핵심 요약

좋은 조직은 AI로 더 빨라지고, 나쁜 조직은 AI로 더 빠르게 망가진다. AI는 **변화의 도구가 아닌 증폭기(accelerator/multiplier)**다. 같은 도구를 쓰는데 어떤 조직은 장애가 2배로 늘고, 어떤 조직은 50% 줄었다. 차이는 도구가 아니라 **기존 조직의 건강도**에 있다.

---

## 1. AI는 조직의 현재 상태를 증폭시킨다

### DX 데이터 (Laura Tacho 키노트, Pragmatic Summit)

- 개발자의 **92%**가 AI 코딩 도구를 월 1회 이상 사용 — 도입은 기정사실
- 주당 평균 **~4시간** 절약 (자기보고 기준)
- AI 도입 후 온보딩 시간도 크게 단축

### 같은 도구, 정반대 결과

| 조직 유형 | AI 도입 결과 |
|-----------|-------------|
| **건강한 조직** | 장애 50% 감소, 코드 품질↑, 유지보수성↑, 변경 속도↑ |
| **불건강한 조직** | 고객 대면 장애 2배 증가, 기존 문제가 더 빠르게 표면화 |

> "기능 장애 조직이 이제 더 빠르게 기능 장애를 일으킨다"

### 핵심 인사이트

AI는 **가속기이자 증폭기**다:
- 건강한 개발 문화 → AI가 품질·속도·유지보수성 개선
- 불안정한 프로세스 + 기술 부채 → AI가 문제를 더 빠르게 드러냄
- 조직의 문제를 "해결"해주지 않는다 — **더 빨리 드러낼 뿐**

## 2. 팀은 작아지고, 속도는 빨라진다

### "피자 두 판 → 피자 한 판" 팀

- 200년 된 농업 장비 회사의 엔지니어링 리더: *"6~10명의 '피자 두 판' 팀이 3~4명의 '피자 한 판' 팀으로 바뀌고 있다"*
- AI와 무관한 전통 제조 기업에서도 일어나는 변화

### Atlassian CTO (Rajeev Rajan)

- 일부 팀이 **코드를 한 줄도 직접 쓰지 않음** — 모두 에이전트/에이전트 오케스트레이션
- 팀 규모가 줄어드는 것이 아니라 **생산성이 2~5배 향상**
- 에이전트가 원격 근무의 새로운 동반자 역할 — 코드 리뷰, 브레인스토밍, 리서치 에이전트

### GitHub 전 CEO (Thomas Dohmke)

- "**AI 네이티브는 새로운 클라우드 네이티브**"
- CTO/CIO가 밤에 에이전트에게 작업 시키고, 아침에 결과 확인하는 패턴 → 조직 전체 변화 추진
- 재미있는 일화: Atlassian CTO가 **자비로 개인 노트북**을 사서 Claude Code 설치 — 회사 IT 정책이 에이전트 도구 설치를 막아서

> "투자자가 대기업이 같은 걸 못하냐고 물으면, Atlassian CTO가 자기 돈으로 노트북 사야 했다고 말하면 됩니다."

## 3. 속도보다 먼저 풀어야 할 것

### Kent Beck, Laura Tacho, Steve Yegge 공동 선언문

> "조직의 문제는 결국 사람과 시스템 수준의 문제다. 그 제약을 먼저 해결하지 않고 기술이 조직 성과를 개선해줄 것이라는 기대에 회의적이다."
>
> "우리는 회의적이고, 우리는 인간이다." — Kent Beck, Laura Tacho, Steve Yegge

### 주요 참석자

Deer Valley 워크숍에 **Martin Fowler, Kent Beck, Steve Yegge, Gene Kim, Laura Tacho, Annie Vella** 등 업계 원로급 인물 50여명 참석. 아무도 "이걸 다 이해했다"고 하지 않았다는 점이 오히려 안심이 되는 대목.

## 4. 나머지 예측들 (원문)

원문(The Pragmatic Engineer)에는 추가 주제도 다루고 있다:

- **미드레벨 엔지니어의 위기**: 신입은 AI 도구에 더 능숙하고, 시니어는 경험이라는 무기가 있음. 중간 경력자가 가장 위협받는 위치
- **리팩토링의 미래**: AI가 코드를 쓰는 시대에도 리팩토링은 여전히 중요 ("hell, yeah!")
- **Agile의 미래**: Agile 선언 25주년, 아이러니하게 Agile 이전의 XP(Extreme Programming) 실천이 부활 중
- **AI 도입 속도**: 과거 모바일/클라우드는 단계적 확산(인디→스타트업→대기업)이었지만, AI는 **모든 기업이 동시에** 움직이는 전례 없는 속도

## 5. MIT 보고서: "The GenAI Divide" 주요 발견

### 보고서 개요

- **MIT Project NANDA** 발행, 2025년 7월
- 300개 이상 AI 이니셔티브 분석, 52개 조직 인터뷰, 153명 시니어 리더 설문

### GenAI Divide란?

기업들의 GenAI 도입률은 높지만, 실질적 비즈니스 전환을 달성한 조직은 극소수인 **양극화 현상**

### 충격적 숫자들

- 기업 GenAI 투자: **$300~400억**
- **95%의 조직이 투자 수익 제로**
- 95%의 맞춤 AI 솔루션이 프로덕션에 도달하지 못함

### 4가지 핵심 패턴

1. **제한적 파괴**: 9개 산업 중 기술·미디어 2개만 구조적 변화
2. **기업의 역설**: 대기업이 파일럿은 가장 많지만 확장 전환율은 가장 낮음
3. **투자 편향**: 예산의 ~70%가 세일즈·마케팅에 집중, 고ROI 백오피스는 과소투자
4. **구현 방식 차이**: 외부 파트너십(66%) vs 내부 구축(33%) — Buy가 Build보다 2배 성공

### 파일럿 → 프로덕션 전환율

| 단계 | 범용 LLM (ChatGPT 등) | 맞춤형 GenAI |
|------|----------------------|-------------|
| 조사 | 80% | 60% |
| 파일럿 | 50% | 20% |
| 성공적 구현 | 40% | **5%** |

### 가장 큰 장벽: "학습 격차(Learning Gap)"

모델 품질·규제·인프라가 아니라 **AI 도구가 학습하지 않고, 맥락을 기억하지 않으며, 시간에 따라 개선되지 않는 것**이 최대 문제.

- 복잡한 업무에서 사용자의 **90%가 AI보다 인간 동료를 선호**
- 이유: 기억력, 적응력, 학습 능력 부족

### Shadow AI 경제

- 공식 LLM 구독 기업: 40%
- 개인 AI 도구 정기 사용 직원: **90%**
- 직원들은 회사 도구를 건너뛰고 개인 ChatGPT를 쓰고 있다

### 인력 영향

- 대규모 해고는 아직 미발생
- 영향 영역: 고객 지원, 행정, 소프트웨어 엔지니어링 (주로 아웃소싱 기능)
- 기술·미디어: 80%+ 경영진이 24개월 내 채용 감소 예상
- 의료·에너지: 채용 감소 예상 없음

### GenAI Divide를 넘는 3가지 원칙

1. **Build보다 Buy** — 외부 파트너십이 2배 성공적
2. **중앙 연구소보다 현장 관리자 권한 부여** — 파워 유저가 도입을 주도하되 경영진 책임 유지
3. **깊이 통합되고 시간에 따라 적응하는 도구** — 정적 도구가 아닌 학습·기억·적응 가능 시스템

## 6. 종합: 가장 균형 잡힌 시각

AI가 조직의 기존 문제를 해결해주지는 않는다. 오히려 그 문제를 **더 빨리 드러낸다**. 이것이 현재 AI 도입을 바라보는 가장 균형 잡힌 시각일지 모른다.

Laura Tacho의 마무리:

> "경이로움과 화성을 향한 야망 사이에서 균형을 찾되, 지구에서 풀어야 할 문제가 있고, 이 현실 속에서 살아야 한다는 것을 이해하길 바랍니다. 근거를 유지하고, 회의적이되, 인간답게, 그리고 무엇보다 실용적으로."

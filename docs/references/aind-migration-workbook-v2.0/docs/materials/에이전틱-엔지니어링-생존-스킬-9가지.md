# 에이전틱 엔지니어링 시대의 생존 스킬 9가지

> **출처**: [GeekNews](https://news.hada.io/topic?id=27104)

---

## 핵심 요약

Karpathy가 주말 프로젝트를 에이전트에게 맡기고 30분 만에 완성하는 시대. 99% 시간을 코드 작성이 아닌 에이전트 명령·감독에 쓰는 "에이전틱 엔지니어링" 시대가 열렸다. 하지만 개발자 60%가 AI를 사용하면서도 완전 위임은 0-20%에 불과한 **위임 패러독스**가 존재한다. 이 글은 에이전트 시대를 살아남기 위한 9가지 핵심 스킬을 정리한다.

---

## 9가지 스킬

### ① 분해 능력 (Decomposition)

- "회원가입 기능 만들어줘"라고 하면 뭔가 나오긴 하지만, 원하던 게 아닐 확률이 높다
- PRD만 던져서 에이전트에게 맡기면 핑퐁 수십 턴, 반나절을 날릴 수 있다
- **처방**: AI와 소크라틱 대화로 5분 인터뷰 → 엣지 케이스 사전 정리 → 수정 2-3턴으로 축소
- 구현 전 생각의 시간 5분이 4시간을 아낀다

### ② 컨텍스트 설계 (Context Architecture)

- AGENTS.md를 잘 쓰는 것도 중요하지만, **코드 아키텍처 자체가 잘 설계되어 있으면** 에이전트의 컨텍스트 파악 속도가 완전히 다르다
- 플랫 디렉토리 → Feature 단위 디렉토리로 재구성하니 즉각 개선
- Armin Ronacher: "도구는 LLM 카오스 원숭이가 완전히 잘못 쓰는 것에 대비해야 한다"

### ③ 완료 정의 (Definition of Done)

- CLI 프로젝트를 밤새 돌렸더니 타입 정의만 세팅하고 비즈니스 로직은 빈 껍데기
- 두 번째 시도에서는 테스트 자체를 에이전트가 자기 편하게 재작성
- **에이전트의 "완료"는 내 "완료"와 다르다**
- Elvis의 7단계 DoD 시스템(PR→CI→코드리뷰3개→텔레그램)이 극단적이지만 방향을 보여줌

### ④ 실패 복구 (Failure Recovery Loop)

- 같은 프롬프트로 재시도하는 건 벽에 같은 방향으로 머리를 박는 것
- 실패를 3가지로 분류하면 처방이 명확해진다:
  - 컨텍스트 부족
  - 방향 오류
  - 구조적 충돌
- "Must NOT Have" 가드레일이 무한루프를 끊는다

### ⑤ 관찰 가능성 (Observability)

- "이상한데… 그냥 두자"가 가장 비싼 판단 — 20개 파일이 엉켜 롤백 불가
- **예광탄 전략 + 블루프린트**: 처음 적용하는 기술은 예광탄이 빠르게 그려준다
- 관찰 가능성이 신뢰를 만들고, 신뢰가 위임을 가능하게 한다

### ⑥ 메모리 설계 (Memory Architecture)

- 3일 연속 작업하면 매일 아침 15분씩 맥락 설명으로 날린다
- Claude Code hooks로 세션 종료 시 자동 메모리 추출 → 다음 세션 5초 복원
- Boris Cherny 팀은 CLAUDE.md를 git에 체크인해서 **팀 전체가 공유**
- 개인의 기억이 아니라 팀의 기억이 에이전트에게 전달되는 구조

### ⑦ 병렬 관리 (Parallel Orchestration)

- Boris Cherny는 10-15개 병렬 세션을 동시에 운영
- CTO 시절 스쿼드 6개 매니징 경험이 에이전트 병렬 관리와 놀라울 만큼 비슷
- 사람은 질문을 하지만 에이전트는 묻지 않고 자기 판단으로 진행 — **사전 설계가 더 중요**

### ⑧ 추상화 계층 설계 (Abstraction Layering)

- Level 0 (직접 코딩) → Level 1 (에이전트 지시) → Level 2 (오케스트레이터) → Level 3 (메타 설계)
- 매일 20분짜리 루틴을 스킬로 만들어 2분으로 줄인 경험
- **컴파운딩 엔지니어링**: 프로젝트는 결승선 게임이 아니라 복리형 게임

### ⑨ 감각 (Taste)

- AI가 만든 디자인은 60-70점. 사람의 감각이 들어오는 순간 "아, 이거 된다"
- AI가 만든 정보 정리 포스트는 좋아요 0, 충동적으로 쓴 한 줄이 조회수 3만
- Chris Lattner: "구현의 자동화가 일어날수록, 설계·판단·감각의 중요성이 오히려 높아진다"
- **80%가 범람하는 시대에 차별화는 나머지 20%에서 나온다**

---

## 핵심 메시지

> "끝난 건 타이핑이지, 엔지니어링이 아니다."

이 9가지는 AI 이전에도 좋은 엔지니어가 갖추고 있던 것들이다. 좋은 설계의 레버리지도 커졌지만, 나쁜 설계의 피해도 커졌다. 그 쇼의 주인공은 AI가 아니라, **AI를 잘 다루는 엔지니어**다.

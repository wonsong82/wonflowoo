# 항상 문제는 '프로세스'다, 바보야

> **출처**: [GeekNews](https://news.hada.io/topic?id=24724)

---

## 핵심 요약

AI는 비즈니스 문제를 스스로 해결하는 마법이 아니다. 비효율적인 프로세스에 AI를 얹으면 "쓰레기를 더 빨리 생산"할 뿐이다. AI 도입 전에 프로세스를 먼저 설계·구조화해야 하며, AI는 그 이후에 가속 도구로 활용해야 한다.

---

## 본문

### AI 전략이 아닌 비즈니스 프로세스 최적화

- 기업들이 "AI 전략"을 논의하지만, 실제로 존재하는 것은 **비즈니스 프로세스 최적화(BPO)** 뿐
- AI는 독립적 전략이 아니라, 이미 존재하는 프로세스를 **가속화하는 도구**
- 비효율적인 구조 위에 AI를 얹으면 문제를 더 빠르게 확대할 뿐

### '마법 지팡이' 착각

- 많은 기업이 AI가 비효율을 자동 제거해 줄 것이라 믿지만, 이는 잘못된 전제
- AI는 지능을 부여하지 않으며, 단지 **결정의 속도만 높임**
- 잘못된 결정을 자동화하면 → "빛의 속도로 어리석은 결정"을 내리는 시스템
- 복잡한 관료적 프로세스에 AI를 적용하면, 직원처럼 불만을 가진 로봇을 만드는 셈

### 비정형 데이터의 함정

- AI는 비정형 데이터(이메일, Slack, PDF, 이미지 등) 처리에 강점을 가진 최초의 기술
- 그러나 이런 데이터에 의존하는 프로세스는 대부분 **비정형적이고 비공식적**
- 고객 불만 처리, 마케팅 기획 등은 문서화되지 않고 숙련자 머릿속에만 존재
- 과거엔 컴퓨터가 처리할 수 없어 사람이 대신했기 때문에, SOP 자체가 없음

### 설계되지 않은 것은 자동화할 수 없음

AI를 적용하려면 먼저 워크플로우를 명확히 설계해야 한다. 세 가지 질문이 필요하다:

1. **트리거**: 비정형 데이터는 어디서 발생하는가?
2. **변환**: 사람이(또는 AI가) 그 데이터에서 무엇을 추출·해석해야 하는가?
3. **출력**: 결과는 ERP나 CRM 같은 구조적 시스템에 어떻게 반영되는가?

### 속도와 지능의 차이

- AI는 더 **빠르게** 만들 뿐, 더 **똑똑하게** 만들지 않음
- 예시: 계약서 50개 검토 → 기존 3일 / AI 3분
  - 프로세스(검토→위험 식별→요약)는 동일, AI가 작동하려면 명확히 정의된 절차 필요
  - '위험'의 의미를 판단하는 **지능적 판단은 여전히 인간의 역할**

### 결론: 프로세스가 전부다

- AI 구세주를 찾는 대신, 화이트보드로 돌아가 가치사슬을 재점검
- 비정형 데이터가 얽힌 인간 중심의 복잡한 영역을 시각화하고 병목·낭비를 찾아야 함
- 프로세스가 단순하고 논리적이며 견고해진 **뒤에야** AI를 가속 장치로 활용 가능
- 기술은 변하지만, **비즈니스 효율의 원칙은 변하지 않음**

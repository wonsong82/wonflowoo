# 에이전트 시대 AI 선택법, 모델보다 하네스가 중요한 이유

> **출처**: [AI Sparkup](https://aisparkup.com/posts/9382) | [원문 (Ethan Mollick, One Useful Thing)](https://www.oneusefulthing.org/p/a-guide-to-which-ai-to-use-in-the)

---

## 핵심 요약

"ChatGPT가 나아요, Claude가 나아요?" — 이제 틀린 질문이다. Wharton 교수 Ethan Mollick은 AI 선택을 **모델·앱·하네스** 세 층위로 봐야 한다고 주장한다. 현재 최상위 모델들의 순수 성능 격차는 점점 줄고 있고, 대부분의 사용자에게는 **하네스(AI가 실제로 무언가를 할 수 있게 만드는 환경)의 격차**가 훨씬 크다. AI는 "말하는 존재"에서 "하는 존재"로 진화했고, 사용자의 역할도 프롬프터에서 관리자로 바뀌고 있다.

---

## 본문

### 1. AI를 고른다는 건 이제 세 가지를 고르는 일

Mollick이 제시하는 세 층위:

| 층위 | 설명 | 예시 |
|------|------|------|
| **모델 (Model)** | AI의 두뇌. 추론·글쓰기·코딩 등 기본 역량 결정 | GPT-5.2, Claude Opus 4.6, Gemini 3 Pro |
| **앱 (App)** | 모델에 접근하는 창구 | chatgpt.com, claude.ai, gemini.google.com |
| **하네스 (Harness)** | 모델의 힘을 실제 작업에 연결하는 장치. 말이 마차를 끌도록 묶는 마구(馬具)에 비유 | 웹 검색, 코드 실행, 파일 처리 등 AI가 실제로 '할 수 있게' 만드는 환경 전체 |

> 똑같은 Claude Opus 4.6이라도 **기본 채팅창**에서 쓸 때와 **Claude Code** 안에서 쓸 때는 완전히 다른 도구다.
> - 전자: 답변을 생성
> - 후자: 수 시간 동안 혼자 소프트웨어를 작성하고 테스트

---

### 2. 모델 성능보다 하네스 격차가 더 크다

- 현재 3대 모델의 순수 성능 차이는 점점 줄어드는 중
- Mollick: **"대부분의 사람에게 모델 차이보다 앱과 하네스가 더 중요하다"**

**실제 사례 — 파일 생성 능력**:
- ChatGPT, Claude: 실제로 작동하는 스프레드시트·PPT 파일 생성 ✅
- Gemini: 문서 파일 자체를 생성하지 못함 ❌
→ 모델 성능이 비슷해도 하네스가 약하면 결과가 달라진다

---

### 3. 강력한 하네스의 사례들

**코딩 에이전트 (최강 하네스)**
- Claude Code, OpenAI Codex, Google Antigravity
- AI에게 코드베이스·터미널·브라우저를 통째로 쥐어주고 "알아서 해결하라"고 맡길 수 있음
- Mollick 사례: GPT-1 가중치 1억 1700만 개를 책으로 만드는 프로젝트 → Claude Code에 맡겨 **약 1시간 만에** 80권짜리 책, 커버 디자인, 웹사이트, 결제 시스템 연동까지 완료

**비개발자용 에이전트**
- **Claude Cowork** (Anthropic, 2026년 1월 출시): 사용자 데스크톱에서 직접 실행. 파일 정리·PDF 데이터 추출 등 계획부터 실행까지 자동 처리
- Mollick: "비기술 업무용 Claude Code" — OpenAI·Google에 직접적인 대응 제품 없음

---

### 4. "말하는 AI"에서 "하는 AI"로

- Mollick은 챗봇 → 에이전트 전환을 **"ChatGPT 등장 이후 AI 사용 방식의 가장 큰 변화"** 로 규정
- 사용자의 역할 변화:
  - 이전: 프롬프트를 정교하게 짜는 사람
  - 지금: AI가 작업을 진행하는 동안 **방향을 잡아주는 관리자**
